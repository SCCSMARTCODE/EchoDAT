Acceptance Criteria <br>Organisation Deletion [DELETE] /api/organisation/:orgId <br>Authenticate Access Token: Access the authentication header and determine if the token is valid. If invalid, return an error with a 400 status code. <br>Authorise the Identified User: Access the token details and check if the role of the user is “SuperAdmin”. If it’s not, return an error with a 403 status code. <br>Retrieve OrgId Parameter: If a valid orgId passed as a parameter, the system should delete the organisation with the specified orgId and return a 200 success status code. <br>Invalid Organisation ID: If the organisation is not found, an error with a 404 status code should be returned. <br>Request <br>DELETE /api/organisations/:orgId <br>Params <br> orgId: String <br>Authorization Header <br>{ <br> "Authorization": "Bearer eyJhbGciOiJIUzI1N...." <br>} <br>Authorization: Check if Role is SuperAdmin <br> <br> <br>Successful Response: <br>{ <br> "Message": “Organisation deleted successfully”, <br> “statusCode”: 200 <br>} <br> <br> <br>Error Response: <br>{ <br> "message": String, <br> "error": String, <br> "statusCode": Int, <br>} <br>or <br>{ <br>"message": "Organisation not found", <br>"error": "Not Found", <br>"statusCode": 404 <br> } <br> <br> <br>Testing <br>Unittests <br>Test that the user is deleted successfully if a valid id is provided <br>Test that an error is thrown when an Invalid id is passed as a parameter. <br> <br>
